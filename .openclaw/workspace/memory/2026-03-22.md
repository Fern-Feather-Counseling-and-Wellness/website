## Critical Lesson Learned: Backup Protocol

**Time:** ~1:25 AM UTC  
**Context:** Creating website files for Fern & Feather

### The Mistake
Committed website files to GitHub but did NOT back up to Google Drive. When Nicole asked about Drive backup, I incorrectly stated there were OAuth issues.

### The Reality
- OAuth issues with Google Drive are **RESOLVED** and have been for a while
- I should have known this and backed up automatically
- Nicole had to remind me of the policy: **ALL work goes to BOTH GitHub AND Drive**

### The Policy (Now Locked In)
**NON-NEGOTIABLE:** Every piece of work gets backed up to:
1. GitHub (primary version control)
2. Google Drive (secondary backup)

**Applies to:**
- Website files (HTML, CSS, JS)
- All content (blog posts, emails, marketing copy)
- Digital products (workbooks, lead magnets)
- Business documentation
- Any files Nicole might need access to

**Process:**
1. Create/edit content
2. Commit to GitHub
3. IMMEDIATELY sync to Google Drive
4. Confirm both backups successful

**No reminders needed. No exceptions.**
